0. RegulatedAuthorization
Invariantesreg-br-001: Registro ANVISA deve seguir formato 1.XXXX.XXXX.XXX-X (identifier.where(system = 'http://anvisa.gov.br/medicamentos/registro').value.matches('^1\.[0-9]{4}\.[0-9]{4}\.[0-9]{3}-[0-9]{1}$'))
reg-br-002: Autorização ativa deve ter período de validade definido (status.coding.where(system = 'http://farmaco.maxapex.net/brig/fhir/CodeSystem/regulated-authorization-status-br' and code = 'active').exists() implies validityPeriod.exists())
reg-br-003: Variações devem ter classificação de variação definida (type.coding.where(system = 'http://farmaco.maxapex.net/brig/fhir/CodeSystem/regulated-authorization-type-br' and code = 'variation').exists() implies extension.where(url = 'http://farmaco.maxapex.net/brig/fhir/StructureDefinition/variation-classification').exists())
reg-br-004: Região deve ser Brasil (region.coding.code = 'BR')
